# Pre-PR Self-Review Checklist

- [ ] The PR title matches what the code actually changes.
- [ ] I ran tests locally and they pass (or I noted why they can’t).
- [ ] No debug artifacts: no print(), breakpoint(), or commented-out code.
- [ ] README/docs are updated if behavior or setup changed.
- [ ] The change is focused (not multiple unrelated changes).
- [ ] I checked formatting/typos and removed unused imports/files.
